Secure Drives - Online Vehicle Rental System
Features and Development Summary
Date: 2026-04-30

1. OVERVIEW
Secure Drives is a comprehensive vehicle rental platform built with Next.js (App Router), PostgreSQL, and Tailwind CSS. It facilitates seamless interaction between vehicle providers and customers, ensuring trust through automated KYC verification and secure payments.

2. CORE FEATURES

A. Authentication and User Management
- Secure user registration and login with bcrypt-hashed passwords.
- Google OAuth integration for quick sign-ins.
- JWT-based authentication with role-based access control (RBAC).
- Three distinct user roles:
  * USER (Customer): Browses and rents vehicles.
  * PROVIDER: Lists and manages their fleet.
  * ADMIN: Oversees the platform, approves vehicles, and manages users.

B. KYC (Know Your Customer) Verification
- Automated document processing using Tesseract.js (OCR) and Sharp (Image Processing).
- Support for extracting data from Citizenship, License, and Passports (English/Nepali).
- Fuzzy name matching to verify user identity against uploaded documents.
- Admin review dashboard for approving/rejecting KYC submissions.

C. Vehicle Management
- Providers can list vehicles with detailed specs (make, model, year, price, etc.) and multiple images.
- Admin-controlled approval workflow: vehicles must be approved before becoming visible to users.
- Availability status tracking.
- Vehicle search and filtering functionality.

D. Booking and Payment System
- Real-time booking process with date range selection.
- Automated price calculation.
- Integrated eSewa Payment Gateway for secure transactions in Nepal.
- Payment status tracking (UNPAID, PAID, REFUNDED).

E. Communication and Notifications
- AI-powered or session-based Chat Support for user assistance.
- Notification system for KYC updates, booking confirmations, and payment status.
- Dashboard for users to manage their bookings and profile.

F. Admin Dashboard
- Centralized management of Users, KYC documents, Vehicles, and Bookings.
- Statistics and reporting (via Recharts).

3. TECHNICAL DEVELOPMENT

A. Tech Stack
- Frontend: React 19, Next.js 15, Tailwind CSS, Lucide React (Icons), Radix UI (Components).
- Backend: Next.js API Routes (Serverless).
- Database: PostgreSQL (using 'pg' pool) with automated schema management (src/lib/db.ts).
- Security: JWT (jose/jsonwebtoken), bcrypt, RBAC middleware.
- Utilities: Sharp (image processing), Tesseract.js (OCR), date-fns (date manipulation), UUID (unique IDs).

B. Architecture
- Modular component-based design in 'src/components'.
- Centralized library for database, auth, config, and utility functions in 'src/lib'.
- Protected routes using Next.js Middleware.
- Responsive design for mobile and desktop compatibility.

4. CURRENT STATUS
- The system is functional with core modules (Auth, KYC, Vehicles, Bookings, Payments) implemented.
- Development focuses on enhancing the user experience, refining OCR accuracy, and expanding provider management features.
