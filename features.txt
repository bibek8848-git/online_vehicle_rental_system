SECURE DRIVES - ONLINE VEHICLE RENTAL SYSTEM
Detailed Project Features and Report Notes

Project Title:
Secure Drives - Online Vehicle Rental System

Project Type:
Full-stack web application for renting vehicles online.

Technology Stack:
- Frontend: Next.js App Router, React 19, TypeScript, Tailwind CSS
- Backend: Next.js API routes
- Database: PostgreSQL
- Authentication: JWT, bcrypt password hashing, Google OAuth support
- UI Libraries: Radix UI, Lucide React icons, Sonner notifications
- Analytics/Charts: Recharts
- OCR/Image Processing: Tesseract.js and Sharp
- Payment Integration: eSewa payment gateway flow

1. PROJECT OVERVIEW

Secure Drives is an online vehicle rental system designed to connect customers with vehicle providers through a secure and managed digital platform. The application allows users to register, verify their identity, browse available vehicles, create booking requests, make payments, and track booking history. It also provides providers with tools to list vehicles, manage bookings, and review fleet performance. An admin panel controls the overall platform by managing users, providers, KYC verification, vehicles, bookings, payments, and platform statistics.

The system is built around three main user roles:
- User/Customer
- Provider/Vehicle Owner
- Admin/System Manager

Each role has a separate dashboard and role-specific permissions. This makes the system organized, secure, and suitable for a real-world vehicle rental workflow.

2. MAIN OBJECTIVES OF THE PROJECT

- To provide a digital platform where customers can rent vehicles easily.
- To allow vehicle owners or companies to list and manage rental vehicles.
- To reduce manual paperwork in vehicle rental processes.
- To increase trust between customers and providers through KYC verification.
- To prevent unauthorized access using role-based authentication.
- To manage booking, payment, and approval workflows from a single system.
- To support secure payment tracking using eSewa.
- To provide an admin dashboard for monitoring and controlling the platform.

3. USER ROLES AND RESPONSIBILITIES

3.1 Customer/User

The customer role is for people who want to rent vehicles. A user can register, log in, complete KYC verification, browse available vehicles, create booking requests, make payments, and view their booking and payment history.

Main user features:
- Register a new account.
- Log in using email/password or Google OAuth.
- Access a user dashboard.
- View KYC verification status.
- Upload identity documents for KYC.
- Browse approved and available vehicles.
- View vehicle details.
- Select rental dates.
- Create a booking request.
- View active and previous bookings.
- Make payment after booking approval.
- View payment history.
- Receive notifications for booking, KYC, and payment updates.
- Use chatbot support for common help topics.
- Manage profile information.

3.2 Provider

The provider role is for vehicle owners or rental businesses. Providers can submit business verification documents, add vehicles, manage their fleet, respond to booking requests, and view analytics related to their rental performance.

Main provider features:
- Register and log in as a provider.
- Access a provider dashboard.
- Upload KYC or business verification documents.
- Add new vehicles with details and images.
- Edit existing vehicle information.
- Delete vehicles from their fleet.
- View vehicle approval status.
- Track pending admin approvals.
- View customer booking requests for their vehicles.
- Approve or reject booking requests.
- Notify customers automatically after booking decisions.
- View fleet statistics.
- View performance analytics such as bookings, revenue, top vehicles, and availability.

3.3 Admin

The admin role is responsible for controlling the entire platform. Admin can view system statistics, manage users and providers, review KYC documents, approve or reject vehicles, track bookings, and monitor payments.

Main admin features:
- Access a protected admin dashboard.
- View total users, providers, bookings, revenue, pending KYC, and pending vehicles.
- Manage registered users.
- View provider accounts.
- Review uploaded KYC documents.
- Approve or reject KYC submissions.
- Add rejection reasons when verification fails.
- Approve or reject provider vehicle listings.
- View all bookings in the system.
- View all payments.
- Monitor platform activity and system health.

4. AUTHENTICATION AND AUTHORIZATION FEATURES

The system includes a secure authentication and authorization flow.

Authentication features:
- Email and password registration.
- Secure password hashing using bcrypt.
- Login API that validates user credentials.
- JWT token generation after successful login.
- Token storage on the client side for authenticated API calls.
- Google OAuth support for users who prefer signing in with Google.
- Middleware-based protection for dashboard routes.

Authorization features:
- Role-based access control.
- Separate dashboard access for USER, PROVIDER, and ADMIN.
- Admin-only APIs are protected.
- Provider-only APIs are protected.
- User-only booking and payment APIs are protected.
- Unauthorized users are redirected to login.
- Users trying to access another role's dashboard are redirected to their own dashboard.
- API requests without valid tokens return proper 401 or 403 responses.

7. KYC VERIFICATION FEATURES

KYC is one of the most important trust-building features in this project. Users and providers can upload identity or business documents, and the system can attempt automatic verification using OCR.

KYC features:
- Upload document type and document image.
- Supports documents such as citizenship, license, passport, and business registration.
- Stores uploaded document data in the database.
- Uses Sharp for image preprocessing.
- Uses Tesseract.js for OCR text extraction.
- Supports English and Nepali OCR language processing.
- Extracts possible name, ID number, date of birth, address, and expiry date.
- Compares extracted name with registered user name using fuzzy matching.
- Automatically approves KYC if name matching succeeds.
- Marks KYC as pending if automatic verification is uncertain.
- Allows admin to manually review KYC documents.
- Allows admin to approve or reject KYC.
- Stores rejection reasons.
- Sends notification after KYC approval or rejection.

KYC status values:
- PENDING
- APPROVED
- REJECTED
- NOT UPLOADED, shown in dashboard when no document exists

Importance of KYC:
- Prevents fake accounts.
- Increases trust between renters and providers.
- Helps ensure only verified users can book vehicles.
- Helps providers verify their business identity.

20. IMPORTANT USER FLOWS

20.1 Customer Rental Flow

1. Customer registers or logs in.
2. Customer completes KYC verification.
3. Customer browses approved vehicles.
4. Customer selects a vehicle.
5. Customer chooses rental dates.
6. System checks booking conflicts.
7. Booking request is created as PENDING.
8. Provider receives booking notification.
9. Provider approves or rejects the booking.
10. Customer receives booking status notification.
11. If approved, customer proceeds to payment.
12. Payment is recorded.
13. Customer can view booking and payment history.

20.2 Provider Vehicle Listing Flow

1. Provider registers or logs in.
2. Provider completes verification.
3. Provider adds a vehicle with details and images.
4. Vehicle is saved as pending approval.
5. Admin reviews the vehicle.
6. Admin approves or rejects the listing.
7. Approved vehicle becomes available for users to browse.
8. Provider receives bookings for approved vehicles.
9. Provider manages requests and tracks analytics.

20.3 Admin Approval Flow

1. Admin logs in.
2. Admin checks dashboard for pending KYC and vehicle approvals.
3. Admin reviews KYC documents and OCR data.
4. Admin approves or rejects KYC.
5. Admin reviews vehicle listings.
6. Admin approves or rejects vehicles.
7. Admin monitors bookings, payments, users, and providers.

23. LIMITATIONS OF THE CURRENT SYSTEM

- Chatbot uses rule-based local responses instead of a fully intelligent AI model.
- Some payment flows may need production eSewa credentials and testing before real deployment.
- OCR accuracy depends on image quality and document format.
- Vehicle location and map-based search could be improved.
- Review system table exists, but full review UI may need expansion.
- Export report button appears in provider analytics UI, but full export functionality may need completion.
- Admin reporting could be extended with downloadable reports.
- Real-time notifications could be improved with WebSocket or polling.

25. CONCLUSION

Secure Drives is a detailed and practical online vehicle rental system that covers the major requirements of a real rental platform. It includes secure authentication, role-based dashboards, KYC verification, provider vehicle management, booking workflows, payment tracking, notifications, chatbot support, and analytics. The system supports three main roles: customer, provider, and admin. Each role has different permissions and responsibilities, making the platform structured and secure.

The project is suitable for a final year project because it combines frontend development, backend API design, database management, authentication, authorization, OCR, payment integration, and dashboard analytics. It solves a realistic problem and can be expanded further into a production-ready rental platform.
